VISÃO DO DIA A DIA
Como é o trabalho de um(a) Engenheiro(a) de Dados:

Construção de Pipelines (ETL/ELT): Criar os "canos" que levam dados de um lugar para outro de forma automática.

Modelagem de Dados: Estruturar tabelas para que os analistas e cientistas encontrem informações rápido.

Monitoramento de Fluxos: Garantir que os dados chegaram no destino sem erros ou perdas.

Qualidade de Dados: Criar travas para que dados "sujos" ou errados não entrem no sistema.

Integração de APIs e Bancos: Conectar diferentes sistemas (vendas, marketing, financeiro) em um único repositório central.

🧠 MAPA DE SKILLS
CORE SKILLS (essenciais):

Python para Dados: Foco em automação e bibliotecas de integração.

SQL Avançado: A linguagem universal dos dados (DDL, DML, Joins, Aggregations).

Estruturas de Dados: Entender como organizar informações para performance.

NICE-TO-HAVE (complementares):

Cloud Foundations: Noções de AWS (S3, Redshift) ou Azure.

Soft Skills de Negócio: Usar sua experiência em vendas para entender o que o "cliente interno" precisa.

FERRAMENTAS E TECNOLOGIAS:

PostgreSQL: Banco de dados relacional para prática.

Apache Airflow: Para orquestração de tarefas (agendar os processos).

Docker: Para criar ambientes isolados de desenvolvimento.

🚀 PROJETO DE PORTFÓLIO
PROJETO: Data Pipeline de Vendas em Tempo Real (Simulado)

O QUE FAZER:
Criar um script Python que simula a chegada de vendas de uma loja (em JSON), limpa esses dados (remove duplicados, padroniza datas) e os carrega automaticamente em um banco de dados PostgreSQL.

ENTREGÁVEIS:

Repositório no GitHub com o código Python organizado.

Um diagrama da arquitetura (desenhado no Excalidraw ou similar).

Um pequeno dashboard em Excel ou Power BI conectado ao seu banco SQL.

CRITÉRIOS DE ACEITAÇÃO:

O pipeline deve rodar sem erros ao ser executado.

Os dados no banco final devem estar formatados corretamente (ex: moedas em decimal).

Deve existir um script SQL para criar as tabelas automaticamente.

DICA: Use sua experiência em vendas para criar "dados fictícios" realistas, como status de pedidos (pago, cancelado, estornado). Isso mostra que você entende o negócio!

💬 ROTEIRO DE ENTREVISTAS
PERGUNTA 1: Como sua experiência em vendas ajuda você como Engenheiro de Dados?
COMO RESPONDER: "Como vendedor, eu entendo que o dado não é apenas um número, é uma decisão de negócio. Sei ouvir as necessidades das áreas e traduzi-las em requisitos técnicos."

PERGUNTA 2: O que é um processo de ETL e qual a parte mais crítica dele?
COMO RESPONDER: "É o processo de Extrair, Transformar e Carregar. A parte mais crítica costuma ser a Transformação, onde garantimos a qualidade e a verdade do dado."

PERGUNTA 3: Qual a diferença entre SQL e NoSQL?
COMO RESPONDER: "SQL é para dados estruturados (tabelas rígidas), excelente para consistência. NoSQL é para dados flexíveis (documentos), ideal para escalabilidade e dados variados."

PERGUNTA 4: Por que versionar código com Git é importante na engenharia?
COMO RESPONDER: "Para garantir que as mudanças no pipeline sejam rastreáveis, permitindo que o time colabore sem apagar o trabalho do outro e facilitando o retorno a versões anteriores se algo quebrar."

PERGUNTA 5: Como você lidaria com um dado que chegou corrompido no pipeline?
COMO RESPONDER: "Implementaria uma 'Dead Letter Queue' ou uma tabela de erros para isolar o dado ruim, notificaria o time e seguiria o processamento dos dados saudáveis."
